# 02. System Structure & Program Execution (시스템 구조와 프로그램 실행)

> **Context:** CPU는 압도적으로 빠르지만 스스로 외부 장치에 접근할 수 없다. 반면 I/O 장치(디스크, 키보드)는 매우 느리다. 만약 CPU가 I/O 작업이 끝날 때까지 기다린다면 시스템은 심각한 자원 낭비를 겪게 된다. 이를 해결하기 위해 등장한 개념이 **인터럽트(Interrupt) 기반 구조**와 **비동기적 처리**다.

## 1. 컴퓨터 시스템의 기본 구조와 보호 메커니즘

CPU는 오직 메모리에 올라온 기계어 명령어(Instruction)만 순차적으로 읽어서 실행할 뿐이다. 

### 왜 제어권 방어 장치가 필요한가?
만약 사용자 프로그램이 무한 루프를 돌며 CPU를 독점하거나, 마음대로 디스크의 중요 파일을 삭제하려고 한다면? 이를 막기 위해 하드웨어적인 방어 장치들이 존재한다.

1. **Mode bit (모드 비트)**
   - **Why:** 운영체제와 사용자 프로그램을 구분하여 시스템을 보호하기 위함.
   - `0 (Kernel Mode)`: 모든 권한을 가짐. 시스템에 중대한 영향을 미치는 보안상 중요한 명령(특권 명령) 수행 가능.
   - `1 (User Mode)`: 제한된 권한. 일반적인 연산만 가능하며, I/O 접근 등의 특권 명령을 실행하려 하면 하드웨어가 차단하고 OS로 제어권을 넘김.
2. **Timer (타이머)**
   - **Why:** 특정 프로그램의 CPU 독점을 막기 위함 (시분할 시스템의 핵심 하드웨어).
   - 정해진 시간이 지나면 타이머가 CPU에 **인터럽트**를 걸어 제어권을 강제로 운영체제(OS)에게 넘기도록 한다.

## 2. 인터럽트(Interrupt)와 시스템 콜(System Call)

현대의 운영체제는 **인터럽트에 의해 구동(Interrupt-driven)**된다. OS는 CPU를 뺏어올 능력이 없으며, 오직 인터럽트가 발생했을 때만 CPU 제어권을 얻어 코드를 실행한다.

- **하드웨어 인터럽트:** I/O 장치(디스크 읽기 완료, 키보드 입력 등)나 타이머가 CPU에 신호를 보내는 것.
- **소프트웨어 인터럽트 (Trap):**
  - **Exception (예외):** 프로그램이 0으로 나누거나, 자신의 메모리 영역을 벗어나는 등 오류를 범했을 때 발생.
  - **System Call (시스템 콜):** 프로그램이 파일 입출력 등 권한이 없는(Mode bit 0이 필요한) 작업을 수행하기 위해 **자발적으로 OS에게 서비스를 요청**하는 인터럽트.

## 3. I/O 수행 방식과 DMA (Direct Memory Access)

CPU가 I/O 컨트롤러에게 "디스크에서 데이터 읽어와"라고 명령(시스템 콜)을 내린 후, 데이터가 올 때까지 어떻게 행동할 것인가?

1. **동기식 입출력 (Synchronous I/O):**
   - I/O 작업이 완료될 때까지 해당 프로그램은 멈춰 대기한다.
   - 이때 CPU까지 같이 쉬게 하면 낭비이므로, OS는 CPU를 I/O와 상관없는 다른 프로그램에게 넘겨주어 일을 시킨다.
2. **비동기식 입출력 (Asynchronous I/O):**
   - I/O 요청만 해두고, 데이터 읽기가 끝나든 말든 해당 프로그램이 곧바로 CPU를 잡고 다른 다음 명령을 계속 수행한다.

### DMA (Direct Memory Access)의 등장 배경
- **Why:** I/O 장치가 데이터를 1바이트씩 읽을 때마다 인터럽트를 걸면, CPU는 원래 하던 일을 멈추고 문맥 교환(Context Switch)을 하는 오버헤드를 너무 많이 겪게 된다.
- **Solution:** 메모리에 직접 접근할 수 있는 컨트롤러인 DMA를 둔다. DMA가 I/O 장치의 로컬 버퍼에서 데이터를 메모리로 블록(Block) 단위로 모아서 한 번에 전송한 뒤, 작업이 모두 끝났을 때만 CPU에 딱 한 번 인터럽트를 건다.

## 4. 프로그램의 실행 (메모리 구조)

프로그램(실행 파일)을 실행하면, 디스크에서 곧바로 물리적 메모리(RAM)로 올라가는 것이 아니다.

1. **가상 메모리 (Virtual Memory):** 각 프로그램은 자신만의 독자적인 `0번지`부터 시작하는 주소 공간(Code, Data, Stack)을 가진다.
2. **주소 변환:** 당장 CPU가 실행해야 하는 핵심 코드만 물리적 메모리에 올려두고, 당장 쓰이지 않는 부분은 디스크의 **Swap Area**에 내려둔다.
3. **커널 주소 공간의 내용:** 운영체제(커널) 역시 하나의 프로그램이므로 가상 메모리에 Code, Data, Stack을 가진다.
   - `Code`: 인터럽트 처리 코드, 시스템 콜 처리 코드, 자원 관리 로직.
   - `Data`: CPU, 메모리 등 하드웨어를 관리하는 자료구조와, 현재 실행 중인 각 프로그램(프로세스)을 관리하기 위한 **PCB(Process Control Block)** 자료구조 저장.
   - `Stack`: 어떤 사용자 프로그램이 시스템 콜을 호출했는지에 따라 각 프로세스별로 별도의 커널 스택을 유지.
