1. 프로세스와 스레드의 차이점을 설명하고, 멀티스레드 환경에서 스레드 간에 공유되는 자원과 그렇지 않은 자원을 구분하여 설명하시오.


- 프로세스는 실행 중인 프로그램 그 자체로, 독립된 메모리 공간(코드, 데이터, 스택, 힙)을 갖습니다. 반면 스레드는 프로세스 내부의 실행 단위로, 하나의 프로세스 안에 여러 스레드가 존재할 수 있습니다.

스레드 간 공유되는 자원은 같은 프로세스의 코드(Code) 영역, 데이터(Data) 영역, 힙(Heap) 영역, 그리고 열린 파일이나 I/O 자원 등입니다. 이를 통해 스레드들은 별도의 통신 없이 데이터를 빠르게 주고받을 수 있습니다.

스레드 간 공유되지 않는 자원은 각 스레드만의 스택(Stack), Program Counter, 레지스터 값입니다. 각 스레드는 독립적인 실행 흐름을 가져야 하므로 이 정보는 반드시 별도로 유지됩니다.

이처럼 스레드는 자원을 최대한 공유하면서도 독립적인 실행 흐름을 유지하기 때문에 프로세스보다 가볍고 효율적입니다.

꼬리질문:
1-1 :  멀티스레드에서 스레드 간 데이터를 공유하기 때문에 발생할 수 있는 문제로서
 Race Condition과 같은 문제가 있죠, 그렇다면,
멀티스레드 환경에서 스레드들이 스택을 공유하지 않는 이유는 무엇인가요?

- 스택은 함수 호출 시 생성되는 지역 변수, 매개변수, 반환 주소를 저장하는 공간입니다. 각 스레드는 독립적인 실행 흐름을 가지므로, 서로 다른 함수를 동시에 호출할 수 있습니다.

- 만약 스택을 공유하면 스레드 A가 함수를 호출할 때 스레드 B의 지역 변수나 반환 주소가 덮어써지는 문제가 발생합니다. 각 스레드가 독립적인 함수 호출 순서(Call Stack)를 유지하려면 스택은 반드시 스레드마다 별도로 존재해야 합니다.

- 반면 힙이나 데이터 영역은 스레드 간 의도적으로 데이터를 공유하기 위한 공간이므로 함께 사용합니다.

2. 동기식 입출력(Synchronous I/O)과 비동기식 입출력(Asynchronous I/O)의 차이를 설명하고, 각각 어떤 상황에서 사용되는지 서술하시오.


- 동기식 입출력(Synchronous I/O)은 I/O 요청을 한 프로세스가 I/O 작업이 완료될 때까지 기다린 후 다음 작업을 진행하는 방식입니다. 구현이 단순하고 순서가 보장되지만, I/O가 완료될 때까지 CPU가 낭비되거나 프로세스가 블록된다는 단점이 있습니다. I/O 결과가 즉시 필요한 경우에 사용됩니다.

- 비동기식 입출력(Asynchronous I/O)은 I/O 요청 후 완료를 기다리지 않고 바로 다음 작업을 진행하는 방식입니다. I/O가 완료되면 인터럽트 등을 통해 프로세스에 알려줍니다. CPU를 효율적으로 활용할 수 있지만, 구현이 복잡하고 결과 처리 타이밍을 별도로 관리해야 합니다.

- 동기식은 파일 읽기처럼 결과가 반드시 필요한 뒤에 진행해야 하는 경우에, 비동기식은 네트워크 통신이나 대용량 데이터 처리처럼 I/O 대기 중에도 다른 작업을 병행해야 하는 경우에 적합합니다.



2-2 꼬리질문 : 동기식 I/O에서 CPU가 낭비되는 것을 막기 위해 운영체제는 어떤 방식을 사용하나요?

- 동기식 I/O에서 프로세스가 I/O 완료를 기다리는 동안 CPU를 그냥 놀리는 것은 매우 비효율적입니다. 운영체제는 이를 막기 위해 두 가지 방법을 사용합니다.

- 첫째, I/O를 요청한 프로세스를 Blocked 상태로 전환합니다. CPU를 잡고 기다리는 것이 아니라 I/O가 완료될 때까지 대기 큐에 넣어두고, 그동안 다른 프로세스가 CPU를 사용하게 합니다.

- 둘째, I/O 완료 시 인터럽트(Interrupt)를 통해 알림을 받습니다. I/O 작업이 끝나면 Device Controller가 CPU에 인터럽트를 보내고, 운영체제는 해당 프로세스를 다시 Ready 상태로 바꿔 스케줄링 대상에 포함시킵니다.

- 이렇게 하면 동기식 I/O이더라도 CPU 낭비 없이 다른 프로세스에 CPU를 활용할 수 있습니다.
