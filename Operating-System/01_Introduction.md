# 1. Introduction to Operating Systems (운영체제 개요)

## 1.1 운영체제의 핵심 목적
운영체제(OS)는 컴퓨터 하드웨어 바로 위에 설치되는 소프트웨어 계층으로서, **컴퓨터 시스템의 자원(CPU, 메모리, 디스크, I/O 등)을 효율적으로 관리**하고 사용자에게 편리한 환경을 제공하는 필수적인 부분이다.

## 1.2 운영체제의 분류 (처리 방식 기준)
가장 중요한 분류 기준은 **처리 방식(Processing)**이다.

- **일괄 처리 (Batch Processing):** 작업이 일정량 모일 때까지 기다렸다가 한 번에 처리한다. (응답 시간이 매우 느려 현대 범용 OS에서는 거의 쓰이지 않음)
- **시분할 (Time Sharing) - 현대 범용 OS의 표준:** 
  - **원리:** CPU의 처리 속도가 인간이 느끼기에 무한히 빠르다는 점을 이용해, 특정 시간(Time Quantum) 단위로 여러 프로그램에 번갈아 CPU를 할당한다.
  - **목적:** 여러 프로그램이 **동시에 실행되는 것처럼(Interactive)** 보이게 하기 위함이다.
- **실시간 (Real-time OS):** 정해진 시간(Deadline) 안에 어떠한 일이 *반드시* 종료됨이 보장되어야 하는 특수 목적 시스템이다. (예: 미사일 제어, 원자로 제어, 자동차 자율주행 등)
  - *Hard Real-time:* 데드라인을 어기면 치명적인 결과 초래 (원자로)
  - *Soft Real-time:* 데드라인을 어겨도 치명적이진 않음 (동영상 스트리밍)

> **💡 [개념 명확화] Multiprogramming vs Time Sharing**
> - **Multiprogramming:** "메모리" 관점. 메모리에 동시에 여러 프로그램이 올라가 있는 상태를 강조. (I/O 대기 시 CPU 낭비를 막기 위한 목적이 큼)
> - **Time Sharing:** "CPU" 관점. CPU의 시간을 잘게 분할하여 나누어 쓴다는 의미를 강조.


## 1.3 운영체제의 핵심 구조 (무엇을 관리하는가)

운영체제의 업무는 결국 **자원별 관리자**로 나뉜다.

- **CPU 스케줄링:** 누구에게 CPU를 줄 것인가? (선착순? 짧은 작업 우선? 시간 분할?)
- **메모리 관리:** 한정된 메모리를 어떻게 쪼개어 프로그램들을 올릴 것인가? 당장 필요한 부분만 올리고 나머지는 디스크(Swap 영역)에 둔다.
- **디스크 스케줄링:** 디스크 헤드가 움직이는 경로를 어떻게 최적화하여 탐색 시간을 줄일 것인가? 엘리베이터의 작동 원리와 유사하다.
- **인터럽트(Interrupt) 및 I/O 관리:** 매우 느린 I/O 장치와 매우 빠른 CPU 사이의 속도 차이를 어떻게 극복할 것인가?
