# 01. Introduction to Operating Systems (운영체제 개요)

> **Context:** 하드웨어는 스스로 판단할 수 없는 깡통에 불과하다. 수많은 소프트웨어가 한정된 하드웨어 자원(CPU, 메모리)을 서로 쓰겠다고 경쟁할 때, 누군가는 교통정리를 하고 악의적인 접근으로부터 시스템을 보호해야 한다. 이 역할을 수행하는 핵심 소프트웨어가 운영체제다.

## 1. 운영체제란 무엇인가?

컴퓨터 하드웨어 바로 위에 설치되어, 사용자 및 다른 모든 소프트웨어와 하드웨어를 연결하는 **소프트웨어 계층**이다.

- **협의의 운영체제 (커널, Kernel):** 컴퓨터 전원이 켜짐과 동시에 메모리에 올라가, 시스템이 종료될 때까지 상주하며 핵심적인 자원 관리를 담당하는 부분. 전공자들이 말하는 OS는 보통 커널을 의미한다.
- **광의의 운영체제:** 커널뿐만 아니라, 각종 시스템 유틸리티(파일 복사 프로그램 등)를 모두 포함한 포괄적 개념.

## 2. 운영체제의 목적 (Why OS?)

운영체제는 두 가지 상충될 수 있는 목표를 동시에 달성해야 한다.

1. **자원의 효율적 관리 (Efficiency & Fairness) - 핵심 목적 = (자원관리자)**
   - **Why:** CPU, 메모리 등의 자원은 한정되어 있지만, 실행하려는 프로그램은 많다.
   - 누구에게 CPU를 먼저 줄 것인가, 한정된 메모리에 어떤 프로그램을 올려둘 것인가를 결정하여 전체 시스템의 성능을 극대화한다. 동시에 특정 프로그램이 차별받지 않도록 형평성(Fairness)도 고려해야 한다.
2. **사용자 편리성 제공 (Convenience)**
   - **Why:** 개발자나 사용자가 복잡한 하드웨어 제어 명령어를 직접 다루는 것은 불가능에 가깝다.
   - OS가 하드웨어의 복잡성을 숨기고(Abstraction), 동시 사용자들에게 마치 혼자 컴퓨터를 독점하는 것 같은 환상(Illusion)을 제공해야 한다.

## 3. 운영체제의 분류 (시분할 시스템의 등장 배경)

가장 중요한 기준은 **처리 방식(Processing)**이다.

- **일괄 처리 (Batch Processing):** 작업이 일정량 모일 때까지 기다렸다가 한 번에 처리. (응답 시간이 느려 현대 범용 OS에서는 거의 쓰이지 않음)
- **시분할 (Time Sharing) - 현대 범용 OS의 표준:** 
  - **Why:** 여러 프로그램이 동시에 실행되는 것처럼 보이게 하기 위함. (짧은 응답시간을 가지기 위해서 시분할 함.)
  - CPU의 처리 속도가 인간이 느끼기엔 무한히 빠르다는 점을 이용해, 시간을 아주 잘게 쪼개어(Time Quantum) 여러 프로그램에 번갈아 CPU를 할당한다. 
- **실시간 (Real-time OS):** 정해진 시간(Deadline) 안에 어떠한 일이 반드시 종료됨이 보장되어야 하는 특수 목적 시스템 (예: 미사일 제어, 원자로 제어).

## 4. 운영체제의 핵심 구조 (무엇을 관리하는가)

운영체제의 업무는 결국 **자원별 관리자**로 나뉜다.

- **CPU 스케줄링:** 누구에게 CPU를 줄 것인가? (선착순? 짧은 작업 우선? 시간 분할?)
- **메모리 관리:** 한정된 메모리를 어떻게 쪼개어 프로그램들을 올릴 것인가? 당장 필요한 부분만 올리고 나머지는 디스크(Swap 영역)에 둔다.
- **디스크 스케줄링:** 디스크 헤드가 움직이는 경로를 어떻게 최적화하여 탐색 시간을 줄일 것인가? 엘리베이터의 작동 원리와 유사하다.
- **인터럽트(Interrupt) 및 I/O 관리:** 매우 느린 I/O 장치와 매우 빠른 CPU 사이의 속도 차이를 어떻게 극복할 것인가?
